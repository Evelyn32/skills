# CBA 原则 —— 访谈题目设计框架

CBA 原则是设计访谈题目的核心框架，确保每个话题从三个角度覆盖，不遗漏信息。CBA 不是用来划分模块的——它是在每个模块内部交叉使用的题目设计方法。

---

## 一、CBA 三要素

### C — Customer / Circumstance（客户 / 情境）

**含义**：还原行为发生时的具体情境和背景。
**目的**：让用户回到真实事件中，而非发表抽象观点。
**典型问法**：
- "当时是在什么情况下……"
- "能描述一下那天的场景吗？"
- "你当时在哪里？和谁在一起？"
- "在______之前，你是怎么做的？"
- "能带我回到你第一次______的那个时刻吗？"

**设计要点**：
- 与 B 类问题形成铺垫关系——先问清楚"当时发生了什么"，再问"你怎么做的"
- 情境问题帮助激活情景记忆（episodic memory），让后续回答更真实
- 不要问得太概括（"你一般什么情况下"），要锚定具体事件（"上一次你______时"）

### B — Behavior / Because（行为 / 原因）

**含义**：用户做了什么选择或行动，以及为什么。
**目的**：了解用户的具体行为和深层动机。
**典型问法**：
- "你是怎么做的？步骤是怎样的？"
- "是什么原因让你选择这个？"
- "你为什么没有选择______？"
- "你是怎么知道这个的？"
- "做这个决定时你最看重什么？"

**设计要点**：
- 行为问题要具体到操作步骤，而非笼统概括
- 原因追问是 B 类最有价值的部分——但不要只停留在"为什么"，尝试 5why 深挖
- B 类问题容易变成引导性问题（"是因为便宜吗？"），保持开放

### A — Attitude / Advice（态度 / 建议）

**含义**：用户怎么看待某个现象、有什么感受、希望怎么解决。
**目的**：获取用户的主观评价和未满足的需求。
**典型问法**：
- "你对这个怎么看？"
- "整个过程下来，你什么感觉？"
- "有没有什么地方让你觉得特别好？有没有让你困扰的？"
- "如果可以改，你希望改成什么样？"
- "你会推荐给别人吗？你会怎么推荐？"

**设计要点**：
- 态度问题放在模块后段——先让用户回忆行为，再评价
- 避免让用户直接给设计方案（"你觉得应该设计成什么样"），而是问"你遇到了什么问题"
- 推荐语是很好的态度探测工具——推荐语反映了用户心中最核心的价值认知

## 二、CBA 在模块内的组织方式

### 标准模式：C → B → A（层层递进）

每个话题模块内的问题按 C→B→A 顺序排列：

```
模块：XXX（约15分钟）

C 类：铺垫进入话题
  → "你第一张银行卡是哪家银行的？"
  
B 类：深挖行为和原因  
  → "当时是什么情况下办了这张卡？""是什么因素让你选这家？"
  
A 类：收束态度和期望
  → "用下来感觉怎么样？""有什么不满意的地方吗？"
```

### 变体：CBA 交叉追问

当话题复杂时，可在一个模块内多次 C→B→A 循环：

```
C1: "你平时怎么管理自己的钱？"（情境打开）
B1: "为什么选择这些方式？"（行为原因）
A1: "你觉得这些方式够用吗？"（态度）

C2: "有没有遇到过什么麻烦？"（引入新情境）
B2: "当时你是怎么应对的？"（行为原因）
A2: "你希望银行在这方面怎么帮你？"（建议）
```

## 三、问题拆解方法

用户很难回答抽象问题，但可以回答具体经历。因此一个研究目标需要拆解为多个具体问题。

### 拆解示例

**研究目标**："探索在校学生使用手机银行的行为模式"

**直接问（无效）**："你是怎么用手机银行的？"

**拆解后（有效）**：
- C："你上一次打开手机银行是什么时候？"（锚定具体事件）
- C："打开之后你做了什么？"（具体行为链）
- B："为什么用手机银行而不是去网点？"（原因对比）
- B："当时有想过用支付宝/微信支付吗？"（竞争方案）
- A："整个操作下来，和你预期的比怎么样？"（态度对比）
- A："如果满分10分你会打几分？扣分扣在哪？"（量化+细节）

### C 类题目模板库

| 目的 | 模板 |
|------|------|
| 打开话题 | "提到______，你第一个想到的是什么？" |
| 锚定事件 | "上一次你______是什么时候？能描述一下当时的情况吗？" |
| 场景还原 | "你当时在哪里？在做什么？旁边还有谁？" |
| 时间线 | "从你想______到最终______，大概经历了多长时间？" |
| 对比情境 | "和平时比，那次有什么不一样？" |
| 引入新话题 | "我们换一个话题——在______方面，你的情况是？" |

### B 类题目模板库

| 目的 | 模板 |
|------|------|
| 行为链 | "你是怎么做的？能一步一步说吗？" |
| 单点原因 | "是什么原因让你选择这个？" |
| 对比原因 | "为什么选A而不是B？" |
| 深层动机 | "这对你来说为什么重要？" |
| 信息渠道 | "你是怎么知道这个的？当时看了哪些信息？" |
| 决策权重 | "做这个决定时，你最看重什么？其次呢？" |
| Switch推力 | "是什么让你觉得之前的方式不够好了？" |
| Switch拉力 | "新方式什么地方让你觉得'这个可以试试'？" |
| Switch焦虑 | "换的时候有没有什么顾虑？" |
| Switch惯性 | "有没有让你想继续用老方法的理由？" |

### A 类题目模板库

| 目的 | 模板 |
|------|------|
| 整体评价 | "用下来整体感觉怎么样？" |
| 亮点挖掘 | "有没有什么地方让你觉得特别好/特别满意？" |
| 痛点挖掘 | "有没有什么地方让你觉得不方便/不满意？" |
| 情绪探测 | "你当时什么感觉？能用一个词来形容吗？" |
| 期望探测 | "如果可以改，你希望改成什么样？" |
| 推荐探测 | "你会推荐给别人吗？你怎么推荐？" |
| 消失测试 | "如果明天这个产品消失了，你会怎么办？" |
| 补充收尾 | "还有什么是你想说但我没问到的？" |

## 五、维度-问题追溯

设计完题目后，需要建立追溯关系，确保每个研究目标都有对应的题目覆盖：

| 研究目标 | 对应模块 | 题目编号 | CBA 类型 |
|---------|---------|---------|---------|
| 探索办卡影响因素 | 模块一：办卡 | Q1-3 | C, B, B |
| 还原消费场景 | 模块二：花钱 | Q4-7 | C, B, A, B |
| 探索理财偏好 | 模块三：生钱 | Q8-11 | C, B, B, A |
| 了解品牌认知 | 模块四：品牌 | Q12-14 | C, A, A |

## 常见误区

- **把 CBA 当成三个模块**：CBA 不是"情境模块、行为模块、态度模块"，而是在每个话题模块内交叉使用
- **C 类问题太概括**："你平时怎么用……"是在问习惯（语义记忆），"上一次你……"才是在问具体事件（情景记忆），后者更真实
- **B 类只停留在表面原因**：问完"为什么"就停了，没有用 5why 深挖
- **A 类让用户给方案**："你觉得应该怎么设计？" → 正确做法是问"你遇到了什么问题"
- **缺少 C 直接问 B**：没有情境铺垫就问"你为什么选这个"，用户只能给理性化的事后解释
