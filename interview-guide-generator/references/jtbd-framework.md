# JTBD —— 决策类话题模块的组织方法

JTBD（Jobs To Be Done）是一种嵌入技术，依附于用户访谈。当研究方案中标注 JTBD 时，它只作用于访谈提纲中**涉及用户选择、切换、决策的话题模块**，而非整场访谈的全部内容。本文件描述如何用 JTBD 的决策链逻辑组织该类模块的题目序列。

---

## 一、核心理念

JTBD 访谈的核心逻辑：**跟着事件走，不跟着观点走。**

当用户开始发表意见和评价时，温和地把话题拉回到"那一次"的具体经历。用户雇用一个产品不是为了产品本身，而是为了完成某个"任务"（job）。竞争不是与同类产品，而是与所有能完成该任务的方式。

```
用户不买电钻，用户买墙上的洞。
用户不办信用卡，用户想要的是"关键时刻能调用的资金"。
```

---

## 二、JTBD 模块的题目序列

在访谈提纲中，一个 JTBD 组织的模块按以下序列推进题目：

### 模块一：开场与热身（5-8分钟）

**目的**：建立信任、了解用户基本画像。
**与后续模块的关系**：为情境重建提供背景信息。

| 问题类型 | 示例 |
|---------|------|
| 身份了解 | "能简单介绍一下你自己吗？你的工作是做什么的？" |
| 产品关系 | "你大概什么时候开始用[产品类别]的？" |
| 使用频次 | "现在用的频率大概是？" |

### 模块二：情境重建（10-15分钟）

**目的**：还原用户第一次/最近一次"雇用"该产品时的完整情境。
**核心原则**：让用户回到那个时刻，而不是概括性回答。

| 探索角度 | 典型问题 |
|---------|---------|
| 时间锚定 | "能帮我回想一下，你上一次做[行为]是什么时候？" |
| 空间还原 | "当时你在哪？在做什么？" |
| 人物关系 | "旁边还有谁？这件事和别人有关吗？" |
| 起点状态 | "在做这件事之前，你的情况是怎样的？" |
| 故事叙述 | "能不能像讲故事一样，从头到尾给我描述一下那天发生了什么？" |

**追问要点**：
- 不要接受概括性回答（"我一般……"），拉回具体事件（"那上次呢？"）
- 捕捉用户的原话和用词，这些词反映了他们的心理模型

### 模块三：Trigger 与寻找过程（10-15分钟）

**目的**：探索从"意识到需求"到"开始寻找方案"的转折过程。
**核心概念**：从被动焦虑到主动行动的转折点。

| 探索角度 | 典型问题 |
|---------|---------|
| 痛点觉醒 | "是什么让你觉得'现在必须做点什么了'？那一天发生了什么？" |
| 问题时长 | "这个问题存在多久了？为什么是现在才想解决？" |
| 被动阶段 | "在你开始找解决方案之前，你忍了多久？怎么忍的？" |
| 转折事件 | "有没有一个具体的时刻或事件，让你终于决定行动？" |
| 寻找启动 | "你最先做了什么？在哪找的？" |
| 信息渠道 | "你都看了哪些信息？问了谁？" |

**追问要点**：
- Trigger 往往是一个微小但具体的事件——不是"有一天我意识到"，而是"那天下午发生了一件事"
- 关注用户用什么词描述"忍不了"的那一刻

### 模块四：竞争方案与决策（10-15分钟）

**目的**：探索用户考虑过的所有替代方案，以及最终选择的决策逻辑。
**核心概念**：竞争方案 ≠ 竞品。竞争方案是"所有能完成该 job 的方式"。

| 探索角度 | 典型问题 |
|---------|---------|
| 方案列表 | "在决定用[产品]之前，你还考虑过哪些其他方式？（包括不用产品的方式）" |
| 淘汰理由 | "那些方式你为什么不选？每一条淘汰的理由是什么？" |
| 选择因素 | "[产品]什么地方让你觉得'这个可以试试'？" |
| 决策权衡 | "有没有什么让你犹豫的地方？差点不选它的理由是什么？" |
| 关键一击 | "最后让你下决心的那一下是什么？" |

**Switch 四力分析（重点追问框架）**：

| 力 | 含义 | 典型问题 |
|----|------|---------|
| **推力** | 离开旧方案的原因 | "是什么让你觉得之前的方式不够好了？" |
| **拉力** | 吸引到新方案的原因 | "新方案什么地方吸引了您？" |
| **焦虑** | 切换的顾虑和阻力 | "换的时候有没有顾虑？最担心什么？" |
| **惯性** | 维持现状的习惯 | "有没有什么东西让你想继续用老方法？" |

### 模块五：使用体验与结果评估（10分钟）

**目的**：了解用户使用后的真实体验，是否完成了最初的 job。

| 探索角度 | 典型问题 |
|---------|---------|
| 使用过程 | "用了一段时间后，和你最初的预期比怎么样？" |
| 完成任务 | "它帮你解决了最开始想解决的问题吗？解决了多少？" |
| 意外发现 | "有没有什么是你没想到的？好的或不好的？" |
| 习惯化 | "现在这个行为变成习惯了吗？有什么变化？" |
| 消失测试 | "如果明天这个产品消失了，你会怎么办？" |
| 改进方向 | "有没有什么地方让你觉得'要是能这样就好了'？" |

### 模块六：收尾（5分钟）

| 环节 | 内容 |
|------|------|
| 查缺补漏 | "我们今天聊了很多，有没有什么是你想说但我没问到的？" |
| 展望 | "未来在这个方面，你觉得会有什么变化？" |
| 感谢 | 表达感谢，说明报酬发放方式 |

## 三、JTBD vs CBA 在提纲中的协作

JTBD 决定了**模块怎么排**（宏观骨架），CBA 决定了**每个模块里的题目怎么写**（中观设计）。

```
JTBD 模块二：情境重建
  ├─ C 类问题："你上一次……当时在哪？"
  ├─ B 类问题："你是怎么做的？为什么？"
  └─ A 类问题："当时你什么感觉？"

JTBD 模块四：竞争方案
  ├─ C 类问题："在考虑用这个之前，你当时用的是什么？"
  ├─ B 类问题："为什么最终放弃了那个？为什么选了这个？"
  └─ A 类问题："换过来之后感觉怎么样？"
```

**判断何时用 JTBD 框架**：

| 研究目标特征 | 推荐框架 |
|------------|---------|
| 涉及用户选择/决策 | JTBD |
| 涉及竞品对比/切换 | JTBD |
| 涉及购买/转化行为 | JTBD |
| 探索需求/痛点/使用习惯 | CBA |
| 验证假设/评估满意度 | CBA |
| 研究方案已给明确话题模块 | 以方案为准 |

## 四、JTBD 问题模板速查

### 情境重建模板
- "能帮我回想一下，你上一次做[行为]是什么时候？当时你在哪？在做什么？"
- "那一天发生了什么，让你觉得需要做这个决定？"
- "能不能像讲故事一样，从那天的早上开始给我描述一下？"

### Trigger 探索模板
- "在你开始找解决方案之前，有什么事情让你意识到'现在必须做点什么'了吗？"
- "这个想法在你脑子里停留了多久才变成行动？"
- "你之前是怎么应对这个问题的？为什么之前能忍、这次不能了？"

### 竞争方案模板
- "在决定用[产品]之前，你还考虑过哪些其他方式？（不用产品也行）"
- "那些方式为什么被你淘汰了？"
- "如果回到当时，你会不会做不同的选择？为什么？"

### Switch 四力模板
- 推力："是什么让你觉得之前的方式不够好了？"
- 拉力："[新方案]什么地方让你觉得'这个可以试试'？"
- 焦虑："换的时候有没有什么顾虑？最让你不安的是什么？"
- 惯性："有没有什么东西让你想继续用老方法的？是什么？"

### 结果评估模板
- "用了一段时间之后，它帮你解决了你最开始想解决的问题吗？"
- "如果明天这个产品消失了，你会怎么办？"
- "和你的预期相比，有什么让你惊喜的？有什么让你失望的？"

## 五、根据研究目标调整重点

| 研究目标 | 重点模块 | 轻点模块 |
|---------|---------|---------|
| 为什么选择我们（而非竞品） | 模块三（Trigger）+ 模块四（竞争方案深挖） | 模块五 |
| 为什么放弃/流失 | 模块二（情境重建）+ 模块三（推力深挖） | 模块四 部分 |
| 发现新 job（探索性研究） | 模块二 + 模块五（结果评估深挖） | 模块四 |
| 产品改进方向 | 模块五（使用体验深挖）+ 模块四（惯性/焦虑） | 模块三 |

---

## 常见误区

- **把竞争方案窄化为同类竞品**：竞争方案是"所有能完成该 job 的方式"——不用任何 App、用 Excel 表格、问朋友，都算
- **跳到用户的解决方案里**：用户说"我希望加一个XX功能"，不要顺着讨论功能，追问"那你想用这个功能完成什么？"
- **接受"因为便宜"这种表面原因**：追问"便宜对你来说意味着什么？便宜让你可以做什么？"
- **JTBD 不是万能框架**：对于不涉及选择/决策的纯粹使用习惯探索，CBA 框架更简洁
