# 问卷生成技能 (Questionnaire Generator)

## 设计目标

根据研究主题和调研目的，自动生成结构完整、题项科学、格式规范的调查问卷。每次使用都应交付一份可直接用于数据收集的完整问卷文档。

## 使用流程

### 输入信息

从体验伙伴 Agent 接收（或用户直接提供）以下信息：

- **研究主题**：调研的核心问题
- **研究目的**：为什么要做这个调研
- **目标人群**：被访者特征（终端用户/专家/内部团队）
- **研究维度**（可选）：已确定的测量构念
- **模型要求**（可选）：如 Kano / ETS / 指定方法论
- **前置研究结论**（可选）：访谈发现、走查结果等

若信息不足，主动追问补齐。

### Step 1: 需求分析与模式识别

#### 1.1 识别设计模式

根据输入判断问卷属于哪种设计模式：

| 设计模式 | 判定条件 | 设计路径 |
|---------|---------|---------|
| **量表驱动型** | 研究方案给出了维度/指标框架，需要量化测量 | 维度→指标→题目，输出测量框架表 |
| **开放探索型** | 目的是发现痛点、收集建议，不预设维度 | 按信息需求分模块设计，开放题为主 |
| **方法论驱动型** | 指定了具体方法论（Kano、ETS 等） | 严格遵循方法论规则设计对应模块 |
| **混合型** | 以上多种模式组合 | 分区设计，各走各自的路径 |

#### 1.2 模型判断（方法论驱动型专用）

解析研究主题，判断适用的问卷模型：

| 信号关键词 | 模型 | 说明 |
|---|---|---|
| "功能优先级""需求分类""功能排序""Kano" | **Kano** | 每个功能点需正反双题 |
| "体验评估""体验诊断""ETS""整体体验水平""体验度量" | **ETS** | 进一步区分 ETS-U / ETS-E |
| "功能满意度""认知调研""行为调研""通用调研" | **通用** | 按量表驱动或开放探索设计 |
| 多目标混合 | **混合** | 组合多个模型/模式模块 |

#### 1.3 ETS 子模式选择（当判定为 ETS 时）

| 条件 | 选择 | 说明 |
|------|------|------|
| 面向终端用户大规模发放 | **ETS-U** | 1 道 NPS + 2 道多选题，2-3 分钟 |
| 专家/内部团队精细评估 | **ETS-E** | 48 条满意度 + 26 条重要度，15-20 分钟 |
| 需要兼顾 | **同时输出两套** | 分别提供 ETS-U 和 ETS-E 问卷 |

#### 1.4 确定后阅读对应参考文件

- Kano → `references/kano-model.md`
- ETS-U / ETS-E → `references/ets-model.md`
- 通用/量表驱动/开放探索 → `references/question-design.md`

### Step 2: 问卷结构设计

#### 2.1 通用结构骨架

所有问卷都遵循以下基本骨架，根据实际需要增减模块：

```
## 一、引言
[研究目的、保密承诺、匿名说明、预计时长、感谢语]

## 二、甄别题（可选）
[筛选目标被访者，标注跳转逻辑]

## 三、主体问卷
### 模块 A：[维度名称]
[题目 + 选项]
### 模块 B：[维度名称]
...

## 四、人口统计学（可选）

## 五、结语
[感谢语、说明]

---

## 附录一：跳转逻辑设置说明
## 附录二：使用说明 & 注意事项
```

#### 2.2 关于甄别题

当调研有特定目标人群时，在引言之后设置甄别题。例如：
- "您是否在过去3个月内使用过本产品？" → 否则结束问卷
- "您的年龄段是？" → 不在目标范围则结束

每道甄别题旁用 `[跳转逻辑：选"否"跳至结语]` 标注跳转条件。

#### 2.3 测量框架表（量表驱动型专用）

当为量表驱动模式时，在输出中生成「维度→指标→操作化定义→对应题目」的映射表：

```
| 维度 | 指标 | 操作化定义 | 对应题目 |
|------|------|-----------|---------|
| [维度] | [指标] | [测量的具体体验] | Q1, Q2 |
```

每个量表类指标至少 2 个题目。每个维度 3-5 个题目为宜。

### Step 3: 题项编写

#### 3.1 题目表述规范

| 规范 | 正确示例 | 错误示例 |
|------|---------|---------|
| 一题一问 | "界面布局清晰易读" | "界面布局和配色都很好看" |
| 具体可感知 | "能快速找到目标功能" | "信息架构合理" |
| 中性表述 | "操作步骤数量是合理的" | "简化了流程，您是否感到满意？" |
| 避免否定嵌套 | "我能顺利完成任务" | "不会因为找不到功能而卡住" |
| 不预设前提 | "遇到问题时能获得有效帮助" | "当产品崩溃时，恢复速度让您满意？" |

#### 3.2 选项设计原则

- **互斥性**：选项之间不重叠
- **穷尽性**：覆盖所有可能，无法穷尽时加"其他（请注明）"
- **平衡性**：正负方向选项数量对称
- **特异性**：避免模糊词，"经常"尽量给出量化定义

#### 3.3 按模型/模式选择题型

- **通用问卷**：单选、多选、Likert 五点/七点、排序、矩阵、开放题
- **Kano 问卷**：每个功能点 1 道正向题 + 1 道反向题，五级选项
- **ETS-U**：1 道 NPS (0-10) + 2 道多选题（8 个选项映射 8 维度）
- **ETS-E**：48 条满意度陈述 (1-5) + 26 个重要度指标 (1-5)

#### 3.4 开放题规范

- 提供思考锚点（"可从效率、功能、界面等角度思考"）
- 避免引导性提问
- 总开放题量不超过 3 题

详细题型规则参见 `references/question-design.md`。

### Step 4: 输出

默认输出 **Markdown 格式**的完整问卷文档。结构如下：

```
# [问卷标题]

## 问卷设计说明（量表驱动/方法论驱动时输出）
### 研究背景与目的
### 设计模式
### 测量框架表（量表驱动型专用）
### 方法论模块说明（Kano/ETS-U/ETS-E）

---

## 一、引言
## 二、甄别题
## 三、主体问卷
## 四、人口统计学
## 五、结语

---

## 附录一：跳转逻辑设置说明
| 题号 | 跳转条件 | 跳转目标 |
|------|----------|----------|

## 附录二：使用说明 & 注意事项
### 适用场景
### 施测方式（线上/线下，推荐工具）
### 填写时长
### 注意事项

## 附录三：数据分析计划
| 分析目的 | 方法 | 涉及题目 | 预期产出 |
|---------|------|---------|---------|

## 附录四：质量控制建议
- 预测试：建议 3-5 人认知访谈
- 无效问卷标准：作答时间 < X 分钟、规律性作答
- 信效度要求：Cronbach's α ≥ 0.7（量表驱动型）
```

如果用户需要 Word 格式，调用 `scripts/generate_questionnaire.py` 将 Markdown 转换为 Word 文档。

### Step 5: 质量检查

生成问卷后逐项检查以下内容：

**题目质量**
- [ ] 是否存在双重问题？（一题问两个概念）
- [ ] 是否存在引导性措辞？
- [ ] 是否存在否定嵌套句式？
- [ ] 是否存在假设性前提？
- [ ] 每个题目能否追溯到研究方案中的具体问题？
- [ ] 开放题是否不超过 3 题？

**结构质量**
- [ ] 量表标度是否统一？（同一含义的标度文字一致）
- [ ] 选项是否互斥且穷尽（MECE）？
- [ ] 多选题是否标注"（可多选）"？
- [ ] 题量是否合理？（目标人群能在规定时间内完成）
- [ ] 跳转逻辑是否完整且正确？

**方法论合规**
- [ ] Kano 模块：正反题是否配对完整？
- [ ] ETS-U：NPS 是否为 0-10 分制？
- [ ] ETS-E：满意度陈述是否完整 48 条？重要度是否 26 项？

**可发放性**
- [ ] 开场说明是否清晰？
- [ ] 题目编号是否连续？
- [ ] 是否可直接导入问卷平台？

## 多角色问卷策略

- **默认**：一套问卷，通过基本信息区分角色
- **角色差异大时**：拆分为多套，核心量表保持一致

## 特殊场景处理

| 场景 | 处理方式 |
|------|---------|
| 研究方案维度不明确 | 自行按 MECE 原则拆解，或走开放探索路径 |
| 有前置访谈数据 | 优先从访谈发现提取维度和题目，标注"来源：访谈" |
| 需要覆盖多个角色 | 默认一套问卷，基本信息区分；差异大时拆分多套 |
| 研究方案同时推荐多种方法 | 按方法论规则设计各模块，自然衔接，控制总题量 |

## 常见错误提醒

- 不要在所有场景下套用同样的结构——先判断设计模式，再差异化设计
- Kano 正反题必须配对，不可遗漏
- ETS-U 的 NPS 是 0-10 分制，不可使用李克特 5 点
- ETS-E 的 48 条陈述必须完整，不可删减
- 附录一（跳转逻辑）、附录二（使用说明）、附录三（数据分析计划）是正式输出的一部分，不可省略
- 量表驱动型必须输出测量框架表，确保每个题目可追溯
