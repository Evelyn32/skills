# Kano 模型问卷设计指南

## 模型概述

Kano 模型用于对用户需求进行分类和优先级排序，将需求属性分为五类：

| 分类 | 含义 | 设计策略 |
|------|------|----------|
| **M** (必备属性) | 必须有的功能，不做会很差，做了理所当然 | 优先满足 |
| **O** (期望属性) | 做的越好越满意，做的差就不满意 | 竞争焦点 |
| **A** (魅力属性) | 令人惊喜的功能，不做无所谓，做了很满意 | 差异化优势 |
| **I** (无差异属性) | 有无都不影响满意度 | 尽量避免投入 |
| **R** (反向属性) | 提供反而导致不满意 | 避免提供 |

此外，**Q** (可疑结果) 表示用户回答矛盾。

## 问卷设计规则

### 双题结构

对**每个功能点**，需要设计一对正反题目：

**正向题（具备时）**：
> 如果 [产品名称] 提供 [功能描述]，您的感受是？
>
> □ 我喜欢这样 □ 我理应如此 □ 我无所谓 □ 我能忍受 □ 我不喜欢

**反向题（不具备时）**：
> 如果 [产品名称] 不提供 [功能描述]，您的感受是？
>
> □ 我喜欢这样 □ 我理应如此 □ 我无所谓 □ 我能忍受 □ 我不喜欢

### 选项含义

| 选项 | 解释 |
|------|------|
| 我喜欢这样 | 让我感到惊喜和满意 |
| 我理应如此 | 这是产品应该具备的基本功能 |
| 我无所谓 | 有或没有我都不在意 |
| 我能忍受 | 不太满意但勉强可以接受 |
| 我不喜欢 | 会让我感到不满 |

## 分类矩阵

将正向题和反向题的回答交叉查询以下矩阵来确定需求属性：

| 正向 \ 反向 | 喜欢 | 理应如此 | 无所谓 | 能忍受 | 不喜欢 |
|-------------|------|----------|--------|--------|--------|
| **喜欢** | Q | A | A | A | O |
| **理应如此** | R | I | I | I | M |
| **无所谓** | R | I | I | I | M |
| **能忍受** | R | I | I | I | M |
| **不喜欢** | R | R | R | R | Q |

### Better-Worse 系数

用于量化功能的优先级排序：

- **Better (满意度增量)** = (A + O) / (A + O + M + I)
- **Worse (满意度减量)** = (O + M) / (A + O + M + I) × (-1)

Better 越高，提供此功能时满意度提升越多；Worse 绝对值越高，不提供此功能时满意度下降越多。

## 编码规则

在数据分析时，将选项转换为数值：

| 选项 | 编码 |
|------|------|
| 我喜欢这样 | 1 |
| 我理应如此 | 2 |
| 我无所谓 | 3 |
| 我能忍受 | 4 |
| 我不喜欢 | 5 |

### 分类判定方法

根据每对正反题的编码组合，查询分类矩阵确定需求属性：

1. 取正向题编码为行、反向题编码为列
2. 在矩阵中定位交叉单元格
3. 该单元格的字母即为需求分类（M/O/A/I/R/Q）

### 质量控制

- 数据清洗：剔除可疑结果——当用户对某个功能的正反题均选"我喜欢这样"或均选"我不喜欢"时，该条回答不可用，记为 Q 并排除
- 可疑型 Q 的占比应 < 10%，超过则说明题目描述可能被误解，需优化措辞后进行重测
- 反向属性 R 占比高的功能点，需确认该功能是否对目标用户确实适得其反
- 当两种属性类型所占比例相近（如魅力型 38%、期望型 35%），可用 Better-Worse 系数辅助判定归属
- Kano 结论具有**即时性**，不能将当前分析结果作为永久依据，产品迭代后应重新评估

### 优先级策略

1. **M (必备属性)** > **O (期望属性)** > **A (魅力属性)** > **I (无差异属性)**
2. 在 Better-Worse 四象限图中：
   - 高 Better 低 |Worse| → 魅力属性（加分项）
   - 低 Better 高 |Worse| → 必备属性（不减分项）
   - 双高 → 期望属性（竞争焦点）
   - 双低 → 无差异属性（可舍弃）

## 问卷结构中 Kano 模块的放置

- Kano 问题一般放在**主体问卷的开头部分**，在甄别题之后
- 每个功能点一组（正向+反向），连续排列
- 在题前说明："以下问题想了解您对各项功能的真实感受，每项功能请回答 A、B 两个小问题"
- 附录中说明 Kano 分类矩阵的计算方式
